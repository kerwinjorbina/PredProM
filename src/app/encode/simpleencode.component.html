Simple Index Encoding

In Simple Index Encoding, we only take the control flow of the traces in the log. This means that we only consider the event name and disregard all other data attributes attached to it. For example, in the given data below, we will not consider the given resources. In any type of index encoding, whether it uses Simple, Latest Payload or Complex, an option is provided if you, as the data scientist, would like to work on specific prefixes (length of the trace to be considered) or the entire log. If you would choose to encode the whole log, the algorithm will take the maximum number of events in all the traces. This will now be the number of columns you will have in your data set. In the event that the trace is shorted than the specified number of columns, we assign a 0 value to it. Otherwise, if a prefix is provided, we will only encode the trace at the given prefix. For example, if the user specified a prefix of 5, we only take the traces which has 5 events.
